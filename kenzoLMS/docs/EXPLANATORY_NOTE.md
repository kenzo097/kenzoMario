# Пояснительная записка

## 1. Суть проекта
Проект Team LMS представляет собой веб-приложение для учебного процесса. Пользователи создают аккаунт, загружают работы и отслеживают их статус.

## 2. Особенности реализации
- Разделение на blueprints (`auth`, `main`, `api`).
- ORM-модели `User` и `Submission`.
- Авторизация через `Flask-Login`.
- Формы и валидация через `Flask-WTF`.
- Загрузка файлов в `app/static/uploads`.

## 3. Почему такой подход
Модульная структура упрощает поддержку и развитие проекта. ORM снижает вероятность ошибок при работе с БД. REST API позволяет интеграцию с внешними сервисами.

## 4. Возможности развития
- Роли (студент/преподаватель).
- Комментарии преподавателя к работе.
- Деплой на хостинг (Render/PythonAnywhere/VPS).
- Интеграция внешнего API (например, анти-плагиат).
